17장.빌트인객체

표준 빌트인 객체

표준 빌트인 객체는 ECMAScript 사양에 정의된 객체를 말하며, 애플리케이션 전역의 공통 기능을 제공한다. 표준 빌트인 객체는 ECMAScript 사양에 정의된 객체이므로 자바스크립트 실행 환경에 관계없이 언제나 사용할 수 있다.

Object, String, Number, Boilean, Symbol, Date, Math, RegExp, Array, Map/Set, Weakmap/WebSet, Fuction, Promise, Reflect, Proxy, Json, Error

호스트 객체

호스트 객체는 ECMAScript 사양에 정의되어 있지 않지만 자바스크립트 실행환경에서 추가로 제공하는 개체를 말한다.

Dom, Bom,, Canvas, XMLHttpRequest, fetch, SVG, Web Copmponent

사용자 정의 객체

사용자가 정의한 객체

문자열, 숫자, 불리언, 값에 대해 객체처럼 접근하면 생성되는 임시 객체를 래퍼 객체라 한다.

eval 함수

eval 함수는 자바스크립트 코드를 나타내는 문자열을 인수로 전달을 받는다. 저달받은 문자열 코드가 표현식이라면 eaval 함수는 문자열 코드를 런타임에 평가하여 값을 생성하고 전달 받은 인수가 표현식이 아닌 문이라면 eval 함수는 문자열 코드를 런타임에 실행한다. 문자열 코드가 여러 개의 문으로 이루어져 있다면 모든 문을 실행한다.

eval 함수는 기존의 스코프를 런타임에 동적으로 수정을 한다.

eval 함수를 통해 사용자로부터 입력 받는 콘텐츠를 실행하는 것은 보안에 매우 취약하다. 또한 eval 함수를 통해 실행되는 코드는 자바스크립트 엔진에 의해 최적화가 수행되지 않으므로 일반적인 코드실행에 비해 처리 속도가 느리다 따라서 eval 함수의 사용은 금지해야한다.

암묵적 전역

foo 함수 내의 y는 선언하지 않은 식별자이다. y = 20이 에러가 날 것 같지만 에러가 나지 않고 식별자 y는 선언이 된 것 처럼 전역 변수처럼 동작을 하는데 이는 선언하지 않은 식별자에 값을 할당하면 전역 객체의 프로퍼티가 되기 때문이다.

y는 변수 선언 없이 단지 전역 객체의 프로피로 추가가 되었을 뿐 변수가 아니므로 호이스팅이 발생 안한다.
