18장.this

동작을 나타내는 메서드는 자신이 속한 객체의 상태 즉 프로퍼티를 참조하고 변경할 수 있어야 한다. 이때 메서드가 자신이 속한 객체의 프로퍼티를 참조하고 변경할 수 있어야 한다. 이때 메서드가 자신이 속한 객체의 프로퍼티를 참조하려면 먼저 자신이 속한 객체를 가리키는 식별자를 참조할 수 있어야 한다.

그것이 this 인데 자신이 속한 객체 또는 자신이 생성할 인스턴스를 가리키는 자기 참조 변수다. this를 통해 자신이 속한 객체 또는 자신이 생성할 인스턴스의 프로퍼티나 메서드를 참조할 수 있다.

this가 가리키는 값, 즉 this 바인딩은 함수 호출 방식에 의해 동적으로 결정된다.

바인딩이란 식별자와 값을 연결하는 과정을 의미한다. 예를 들어 변수 선언은 변수 이름과 확보된 메모리 공간의 주소를 바인딩 하는 것이다. this 바인딩은 this(키워드로 분류되자만 식별자 역할을 한다)와 this가 가리킬 캑체를 바인딩 하는 것이다.

자바스크립트의 thus눈 함수가 호촐되는 방식에 따라 this에 바인딩 도리 값 즉 this 바인딩이 동적으로 결정된다.

this는

  • 일반 함수로서 호출 - 전역객체 가르킴
  • 메서드로 호출 - 메서드를 호출한 객체(맞춤표의 앞에 객체) >> 즉 화살표 함수에서도 위의 객체를 가르킨다. 화살표 함수에는 this라는 변수 자체가 존재하지 않기 때문에 그 상위 환경에서의 this를 참조하게 됩니다
    • 무조건 익명함수로만 사용할 수 있음
    • 메소드나 생성자 함수로 사용할 수 없음 > 어짜피 메소드에서 this는 상위 객체를 가르킴
  • 생성자 함수로서 호출 - 생성자 함수가 미래에 생성할 인스턴스
  • 이벤트 핸들러에서는 이벤트가 발생한 곳을 가르킨다. // 여기서 this를 사용할 경우 콜백함수에 화살표 함수를 안쓰는 것이 좋다.
  • 객체 프로퍼티에 저장된 함수를 '메서드’라고 부릅니다.
  • 메서드는 this로 객체를 참조합니다
