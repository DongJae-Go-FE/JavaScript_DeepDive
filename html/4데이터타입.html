4장. 데이터 타입

데이터 타입은 값의 종류를 말한다.

자바스크립트는 7개의 데이터 타입을 가지고 테이터 타입은 크게 원시 타입과 객체 타입으로 분류할 수 있다.

원시 타입 - number, string, boolean, undefined, null, symbol

객체 타입 - object, function, symbol

자바스크립트에서 1 과 "1"은 다르다

이렇게 나누는 이유가 무엇일까?

자바스크립트에서는 데이터 타입에 따라 데이터를 저장해야할 메모리 공간의 크기도 다르고 메모리에 저장되는 2진수도 다르고 해석하는 것도 다르다. 그래서 타입의 구별이 필요

number(숫자 타입)

c나 자바는 정수와 실수를 구별하지만 자바스크립트는 하나의 숫자 타입으로 지정하며 모두 실수로 처리, 그리고 2진수, 8진수, 16진수를 표현하는 데이터 타입을 제공 안하기 때문에 모두 10진수로 해석된다.

console.log(1 === 1.0) // true

숫자 타입은 추가적으로 세 가지 특별한 값도 표현 가능

Infinity : 양의 무한대

-Infinity : 음의 무한대

NaN : 산술 연산 불가 > 대소문자 구별 중요

string(문자 타입)

문자열 타입은 텍스트 데이터를 나타내는데 사용

"", ''. ``으로 텍스트를 감싼다.

var string

string = '문자열';

string = "문자열";

string = `문자열`; es6 백틱

템플릿 리터럴

``을 사용해서 텍스트를 변수에 할당하면

` 이런식으로 개행도 가능 `

표현식 삽입

var first = 'kar'; console.log('My Name Is' + first)

문자열 연산자 +를 통해 연결할 수 있다.

console.log(`my name is ${first}`)

백틱을 사용하고 백틱 안에 ${} 사용해서 변수 같은 값을 넣을 수 있다.

boolean(불이언 타입) 타입

값이 참과 거짓을 판별할 때 사용 프로그램의 흐름을 제어하는 조건문에서 자주 사용

true & false 밖에 없다.

undefined

편하게 생각을 하자면 변수를 비워둘 수 없으니 값을 할당하여 변수의 실체를 명확히 하는 것을 말한다.

undefined 값이 유일하다.

var 키워드로 선언한 변수는 암묵적으로 undefined 초기화된다. 다시 말해 메모리에 할당이 되기 까지 전까지는 garbege 값이 들어 있고 선언을 하면 내버려 두지 않고 undefined로 초기화 한다.

그렇다고 변수를 초기화 하는데 undefined로 사용하는 권장하지 않고 null을 권장한다.

null

null 타입은 null이 유일하다. 대소문자 구분 중요!

null은 변수에 값이 없다는 것을 의도적으로 명시할 때 사용하고 변수가 이전에 참조하던 값을 더이상 참조하지 않겠다는 의미

symbol (심벌 타입)

심벌 타입은 es6에서 추가된 7번째 타입으로 변경 불가능한 원시 타입 값 심벌 값은 다른 값과 중복되지 않는 유일 무이한 값

object (객체 타입)

자바스크립트 타입은 크게 원시 타입과 객체 타입으로 분류를 한다고 했다

이것만 기억을 해라 자바스크립트는 객체 기반 언어이며, 자바스크립트를 이루고 있는 거의 모든 것이 객체라는 것

데이터 타입의 필요성

데이터 타입은 왜 필요할까?

메모리에 값을 저장하려면 먼저 확보를 해야할 메모리 공간의 크기를 결정해야 하고 몇 바이트의 메모리 공간을 사용해야 낭비와 손실 없이 값을 저장할 수 있는지 알아야한다.

크게 정리를 하자면

값을 저장할 때 확보해야 하는 메모리 공간의 크기를 결정하기 위해

값을 참조할 때 한 번에 읽어 들여야 할 메모리 공간의 크기를 결정하기 위해

메모리에서 읽어 들인 2진수를 어떻게 해석할지 결정하기 위해

동적 타이핑

c나 자바 같은 정적 타입 언어는 변수를 선언할 때 변수에 할당할 수 있는 값의 종류, 즉 데이터 타입을 사전에 선언해야하지만 자바스크립트는 그런 것이 없다

그래서 정적 타입 언어는 컴파일 시점에 타입 체크를 한다.

그러나 자바스크립트는 선언 키워드만 있지 따로 타입을 체크하는 것이 없어 typeOf 연산자로 체크를 한다.

자바스크립트의 변수는 선언이 아닌 할당에 의해 타입이 결정(타입 추론)이 된다. 그리고 재할당에 의해 변수 타입은 언제든지 동적으로 변할 수 있다. 이러한 특징을 동적 타이핑이라 하고

결론은 자바스크립트는 동적 타입 언어라고 한다.
