13장.let, const 키워드와 블록 레벨 스코프

es5까지 자바스크립트의 변수를 선언할 수 있는 유일한 방식은 var 밖에 없었지만 es6부터는 let const가 추가

var 키워드

  • 중복선언 가능
  • 재할당, 재선언 가능
  • 호이스팅 발생 - 선언단계와 초기화 단계가 동시에 발생
  • 함수의 코드 블록만을 지역 변수로 여긴다 즉 블록 레벨 스코프는 지역 변수로 여기질 않는다.
  • 이제는 잘 쓰지 않으면 좋겠다
  • 전역 변수용

let 키워드

  • let 키워드로 선언을 하면 동일한 변수명을 가진 변수를 중복 선언이 금지가 된다.
  • 재할당은 가능하다
  • 블록 레벨 스코프에서도 지역 스코프로 인정을 한다.
  • 호이스팅이 일어나질 않는다. 선언단계와 초기화 단계가 분리되어 진행된다. 만약 초기화 단계 이전에 변수에 접근을 하려고 하면 오류가난다
  • let 키워드로 선언한 변수는 스코프의 시작 지점부터 초기화 단계 시작 지점(변수 선언문)까지 변수를 참조할 수 없다. 스코프 시작부터 초기화 시작 지점까지 변수를 참조할 수 없는 구간을 일시적 사각지대라고 한다.
  • 전역 객체 내에서 let 선언을 하면 window전역 객체 내에서 접근 불가

const 키워드

  • const 키워드로 선언한 변수는 선언과 동시에 초기화를 해야한다.
  • const 키워드로 선언한 변수는 재선언 재할당 다 금지다
  • const는 상수이므로 상수 === 재할당이 금지된 변수를 말한다.
  • const 키워드로 선언된 변수에 원시 값을 할당한 경우 원시 값은 변경할 수 없는 값이고 const 키워드에 의해 재할당이 금지되므로 할당된 값을 변경할 수 있는 방법은 없다.
  • 하지만 const 키워드로 선언된 변수에 객체를 할당한 경우 객체 값을 변경할 수 있다. 이는 const 키워드는 재할당을 금지할 뿐 불변을 의미하지 않는다.
